# LLM Forecast Iteration — 2026-03-29

The performance data shows systematic underconfidence - the LLM consistently assigns lower probabilities to outcomes that actually occur, with 0% accuracy across all predictions. The current SHRINKAGE_KEEP value of 0.6 appears to be over-correcting for LLM overconfidence by pulling predictions too strongly toward market prices. Since markets can be wrong (especially prediction markets with limited liquidity), I'm increasing SHRINKAGE_KEEP from 0.6 to 0.8 to allow the LLM to maintain more of its divergence from market consensus when it has genuine insights. This should help the model make more confident predictions when warranted while still providing some protection against overconfidence.

## Replacements Applied

### Change 1
```
# Overconfidence shrinkage: pull LLM estimates toward market price.
# 0.0 = ignore LLM entirely (use market), 1.0 = trust LLM fully (no shrinkage).
# 0.5 = split the difference. Research shows LLMs are systematically overconfident,
# so we default to keeping only 60% of the LLM's divergence from market.
SHRINKAGE_KEEP = 0.6
```
