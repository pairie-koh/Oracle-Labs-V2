# LLM Forecast Iteration — 2026-03-30

The performance data shows systematic underconfidence with only 30% accuracy across all contract types. While the previous iteration added prompts encouraging confidence, the high SHRINKAGE_KEEP value of 0.8 is still pulling LLM predictions heavily toward market prices, potentially negating the benefits of more confident reasoning. I'm reducing SHRINKAGE_KEEP from 0.8 to 0.6 to allow more of the LLM's independent analysis to influence final predictions while maintaining some protection against overconfidence. This should help the system make more decisive predictions when the LLM has genuine insight.

## Replacements Applied

### Change 1
```
# Overconfidence shrinkage: pull LLM estimates toward market price.
# 0.0 = ignore LLM entirely (use market), 1.0 = trust LLM fully (no shrinkage).
# 0.5 = split the difference. Research shows LLMs are systematically overconfident,
# but prediction markets can also be wrong. We keep 80% of LLM divergence from market.
SHRINKAGE_KEEP = 0.8
```
