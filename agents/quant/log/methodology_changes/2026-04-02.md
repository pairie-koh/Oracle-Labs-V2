# quant — 2026-04-02

Version: 1.13.0 -> 1.14.0

Removed the adaptive volatility-based blending mechanism that was potentially causing directional errors. Returned to using the fixed BASE_MOMENTUM_BLEND parameter of 0.5 instead of computing adaptive_blend from volatility ratios. This simplifies the model and should improve directional accuracy since the volatility-based logic may have been systematically biasing predictions in the wrong direction. The approach now uses a straightforward 50/50 blend between momentum and mean reversion.
