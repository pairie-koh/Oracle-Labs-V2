# Quant Methodology Change — 2026-03-16

**Version:** 1.2.0 → 1.3.0

## Change

Reduced the reversion forecast pull coefficient from `0.1` to `0.05`.

```python
# Before
reversion_forecast = current + (long_mean - current) * 0.1

# After
reversion_forecast = current + (long_mean - current) * 0.05
```

## Reasoning

Yesterday's change (reducing the momentum pull from 0.5 → 0.15) was directionally correct — the most recent per-cycle MSE (0.00230) is the best in the last 5 cycles. Overall MSE (0.00471) remains ~58× worse than naive baseline (0.000080), meaning both the momentum and reversion terms are still introducing too much deviation from current price. In this stable low-volatility regime, the naive baseline wins simply by predicting no change. The reversion term was pulling forecasts toward a long-term mean that can differ significantly from current price (the market has been declining over time), so halving the reversion coefficient from 0.1 to 0.05 further dampens that over-correction while preserving a weak directional signal.
